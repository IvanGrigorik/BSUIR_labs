КОНТРОЛЬНАЯ РАБОТА

 

Основная цель выполнения контрольной работы состоит в закреплении самостоятельно изученного в межсессионный период учебного материала путем выполнения конкретного задания.

Варианты контрольной работы соответствуют лабораторным работам, выполняемым студентами дневной формы обучения, за исключением тех лабораторных работ, которые проводятся как таковые и со студентами заочной формы обучения.

Допускается выбор варианта самим студентом.

Вариант контрольной работыСодержание
Программная передача сообщений между двумя станциями посредством COM-портов (на базе эмулятора).Написание простейшей коммуникационной программы, позволяющей передать сообщение посредством двух соединенных между собой COM-портов персональных компьютеров. Лучше всего выполнять на базе программных эмуляторов COM-портов (например, Eterlogic). Язык программирования -- по своему выбору.
Программная реализация пакетной передачи и алгоритма бит- либо байт-стаффинга.Написание соответствующей программы на одном из языков программирования по своему выбору.
Программная реализация кода CRC либо Хэмминга.Написание соответствующей программы на одном из языков программирования по своему выбору.
Программная реализация упрощенного варианта алгоритма CSMA/CD.Написание соответствующей программы на одном из языков программирования по своему выбору.
Программная реализация упрощенного варианта алгоритма Token Ring.Написание соответствующей программы на одном из языков программирования по своему выбору.

Лабораторная работа на тему «Обжим витой пары» по понятным причинам не может рассматриваться как контрольная работа.

Отчет о контрольной работе должен быть оформлен в соответствии с действующими кафедральными требованиями и должен содержать ключевые фрагменты написанного программного кода.
